# Append Performance Experiment Log

This file records every append-performance experiment Kiroku has run. It is
**append-only by convention**: new experiments are added as new rows at the
bottom of the table, prior rows are not edited even when later evidence changes
how we interpret them. To revise a prior finding, append a new row whose
**Lesson** column points at the earlier row's date and explains the
reinterpretation. The convention is enforced by code review, not by tooling.
The discipline for proposing a new experiment lives in
[`docs/PERF-METHODOLOGY.md`](PERF-METHODOLOGY.md).

Each row records:

- **Date** — when the experiment was run (ISO `YYYY-MM-DD`).
- **Plan** — full repository-relative path to the ExecPlan that introduced the
  experiment.
- **Hypothesis** — one sentence stating what was expected to improve and why.
- **Variant** — what was actually tested (SQL shape, dispatch path, schema
  change, benchmark-only probe, informal trial).
- **Bench Numbers** — the bench cells and values that supported the
  keep / revert / not-implement decision. Quote figures verbatim from the
  source plan; mark `(qualitative, no CSV)` when no CSV exists.
- **Outcome** — one of `kept`, `reverted`, `not-implemented`.
- **Lesson** — a one-line takeaway.
